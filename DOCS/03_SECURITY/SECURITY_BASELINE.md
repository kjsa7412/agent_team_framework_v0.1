# 보안 기준선 (Security Baseline)

모든 에이전트가 반드시 준수해야 하는 최소 보안 기준.

## 필수 보안 규칙

### 1. 시크릿 관리
- API 키, 비밀번호, JWT 시크릿을 코드에 하드코딩 금지
- 환경변수(.env)는 절대 커밋하지 않음
- .gitignore에 반드시 .env 포함

### 2. 인증/인가
- 모든 API 엔드포인트에 인증 적용 (공개 API 제외)
- 공개 API는 명시적으로 허용 목록 관리
- 권한 없는 접근은 401/403으로 처리

### 3. 입력 검증
- 모든 사용자 입력 검증
- SQL 인젝션 방지: MyBatis 파라미터 바인딩 사용 (#{}), ${} 사용 금지
- XSS 방지: 사용자 입력을 화면에 출력 시 이스케이프 처리

### 4. 로그 보안
- 비밀번호, 토큰, 개인정보를 로그에 출력 금지
- 로그에는 사용자 ID만 포함 (이메일, 이름 등 개인정보 제외)

### 5. HTTPS
- 운영 환경에서 HTTPS 필수
- HTTP → HTTPS 리다이렉트 설정

### 6. CORS
- 허용된 출처만 CORS 설정
- 운영 환경에서 와일드카드(*) 사용 금지

## 보안 위반 처리

보안 위반 발생 시:
1. 즉시 오케스트레이터에게 보고
2. 해당 Work Item BLOCKED 처리
3. 보안 수정 후 재검토 필요

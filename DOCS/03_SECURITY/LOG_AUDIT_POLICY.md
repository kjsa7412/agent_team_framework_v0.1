# 로그/감사 정책 (Log & Audit Policy)

## 로그 원칙

1. 에러 로그는 반드시 남김
2. 개인정보를 로그에 포함하지 않음
3. 비밀번호/토큰을 로그에 출력하지 않음

## 로그 레벨

| 레벨 | 사용 시점 |
|------|----------|
| ERROR | 시스템 오류, 예외 발생 |
| WARN | 경고, 비정상 상황 |
| INFO | 주요 비즈니스 이벤트 |
| DEBUG | 개발/디버깅용 (운영 제외) |

## 감사 로그

- 관리자 작업은 반드시 감사 로그 기록
- 인증 이벤트 (로그인 성공/실패) 기록
- 중요 데이터 변경 기록

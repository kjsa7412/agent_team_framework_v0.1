# 정책 우선순위 (Policy Priority)

정책 충돌 시 아래 순서로 우선순위를 적용한다.

## 우선순위 (높음 → 낮음)

1. **03_SECURITY** — 보안 정책은 최우선
2. **01_GOVERNANCE** — 거버넌스 정책
3. **09_WORKFLOW** — 워크플로우 정책
4. **04_DEVELOPMENT** — 개발 표준
5. **02_ARCHITECTURE** — 아키텍처 정책
6. **05_QUALITY** — 품질 정책
7. **06_OPERATIONS** — 운영 정책
8. **07_PRODUCT** — 제품 정책
9. **08_INTELLIGENCE** — 인텔리전스 정책
10. **90_ADAPTIVE** — 적응형 정책 (검증 중)

## 원칙

- 보안 정책은 어떤 이유로도 타협하지 않음
- 정책 충돌 발생 시 오케스트레이터가 관리자에게 보고
- 적응형 정책(90_ADAPTIVE)은 정식 승격 전까지 가이드라인으로만 사용

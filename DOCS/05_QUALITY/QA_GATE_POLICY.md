# QA 게이트 정책 (QA Gate Policy)

## QA 게이트 정의

릴리즈 전 반드시 통과해야 하는 품질 검증 관문.

## 게이트 항목

| 항목 | evolution-small | evolution-large | foundation-rebuild |
|------|:--------------:|:---------------:|:-----------------:|
| 단위 테스트 | 필수 | 필수 | 필수 |
| 변경 기능 시나리오 | 필수 | 필수 | 필수 |
| 회귀 테스트 | 핵심만 | 전체 | 전체 |
| 통합 테스트 | 선택 | 필수 | 필수 |
| 스모크 테스트 | 선택 | 선택 | 필수 |

## 게이트 통과 조건

1. 선택된 시나리오의 모든 테스트 케이스 통과
2. 블로킹 이슈 없음
3. 회귀 없음 확인
4. test-orchestrator 승인

## 게이트 실패 시

1. 실패 항목을 outputs/test/failures.md에 기록
2. 해당 Work Item BLOCKED 처리
3. 구현 그룹에 수정 요청
4. 수정 후 재검증

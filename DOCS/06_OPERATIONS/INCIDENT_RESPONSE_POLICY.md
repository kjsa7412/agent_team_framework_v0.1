# 장애 대응 정책 (Incident Response Policy)

## 장애 단계 정의

| 단계 | 설명 |
|------|------|
| P1 | 서비스 전체 중단 |
| P2 | 핵심 기능 장애 |
| P3 | 일부 기능 이상 |
| P4 | 경미한 이슈 |

## 대응 절차

### P1/P2 장애
1. 즉시 관리자 보고
2. 롤백 또는 핫픽스 결정
3. 롤백 실행 (ROLLBACK_POLICY.md 참조)
4. 원인 분석
5. 수정 및 재배포
6. 장애 보고서 작성

### P3/P4 장애
1. 관리자 보고
2. Work Item 생성 (evolution-small 또는 fix)
3. 정상 배포 프로세스로 수정

## 장애 기록

모든 P1/P2 장애는 반드시 기록:
- 발생 시각
- 영향 범위
- 원인
- 대응 내역
- 재발 방지 방안

# 제품 범위 정책 (Product Scope Policy)

## MVP 범위 정의 원칙

MVP = 운영 가능한 최소 기능 집합

### 포함 기준
- 핵심 사용자 가치를 제공하는 기능
- 서비스 운영에 필수적인 기능
- 보안/인증 기본 기능

### 제외 기준
- "있으면 좋은" 기능
- 운영 후 데이터 기반으로 결정할 기능
- 복잡도 대비 가치가 낮은 기능

## 범위 변경 정책

- MVP 범위는 planning-agent가 정의
- 범위 확장은 Work Item으로 관리
- 범위 축소는 관리자 승인 필요

## 우선순위 체계

| 우선순위 | 설명 |
|---------|------|
| Must | 반드시 있어야 함 |
| Should | 있어야 하지만 없어도 동작 가능 |
| Could | 여유가 있으면 추가 |
| Won't | 이번 MVP에서 제외 |

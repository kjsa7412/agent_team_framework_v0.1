# 에이전트 실행 정책 (Agent Execution Policy)

## 에이전트 실행 원칙

1. 모든 에이전트는 작업 전 관련 DOCS 문서를 읽음
2. 오케스트레이터 승인 없이 독단적 실행 금지
3. 각 에이전트는 자기 완료 조건(AC)을 명확히 알고 시작
4. 불명확한 요구사항은 추정하지 않고 오케스트레이터에게 질의

## 에이전트 내부 문서

각 에이전트 폴더 내 필수 문서:
- README.md: 역할 및 이번 작업 개요
- execution.md: 실행 계획 및 진행 기록
- inputs.md: 입력 문서 목록
- outputs.md: 산출물 목록
- handoff.md: 다음 에이전트에게 넘길 내용
- result.md: 최종 결과 요약

## 실행 순서

```
오케스트레이터 분석 → 에이전트 할당 → 에이전트별 실행 → Handoff → 다음 에이전트
```

## 금지 사항

- 오케스트레이터 없이 에이전트 간 직접 통신
- 승인 없이 보안 정책 위반
- AC 미충족 상태로 DONE 표시

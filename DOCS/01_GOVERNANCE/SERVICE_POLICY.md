# 서비스 운영 정책 (Service Policy)

## 핵심 원칙

### 원칙 1. 전역 정책과 개별 작업 분리
- 전역 정책: DOCS/
- 개별 작업: docs/work-items/
- 에이전트 실행 정의: .claude/

### 원칙 2. Work Item 동적 구조
- 고정 단계형 폴더 구조 사용하지 않음
- 실제 투입된 에이전트 중심으로 구성

### 원칙 3. 규모가 아니라 변화의 성격으로 라우팅
- 대규모 패치라도 기존 구조 위에서 처리 가능 → Evolution
- 구조를 다시 세워야 함 → Foundation 재호출

### 원칙 4. 테스트 독립 조직
- Test Group은 Foundation/Evolution의 하위가 아님
- 독립 조직으로 운영

### 원칙 5. 적응형 프레임워크
- 현재 구조를 정답으로 고정하지 않음
- 운영 중 보정점은 DOCS/90_ADAPTIVE/에 누적

## MVP 정의

MVP = 운영 가능한 최소 구조를 갖춘 상태
- 핵심 기능 동작
- 인증/권한 작동
- 기본 배포 가능
- 최소 테스트 통과
